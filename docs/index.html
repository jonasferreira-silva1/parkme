Problema & Solução

Por que o ParkMe existe

Você chega em um shopping lotado, anda em círculos por 10 minutos procurando uma vaga, finalmente encontra uma — e quando volta, passa mais 15 minutos tentando lembrar em qual setor deixou o carro. O ParkMe resolve isso.

Entrada registrada pelo operador → vaga atribuída automaticamente → motorista sabe exatamente onde ir.

Enviado para o motorista via room user_{userId}. Gatilho manual ou por job futuro.

Broadcast periódico com percentual de ocupação do estacionamento.
{ "total": 120, "ocupadas": 97, "percentual": 80.8 }

import { io } from 'socket.io-client';

const socket = io('http://localhost:3000/parking', {
  auth: { token: getJwtToken() },  // JWT validado no handshake
});

// Entra na sala do estacionamento

// Atualiza o mapa quando vaga muda

Regras de Negócio

Algoritmos, fórmulas e transições de estado

1. Atribuição Automática de Vagas

O algoritmo escolhe a melhor vaga disponível seguindo a ordem de prioridade:
// Pseudocódigo — atribuirMelhorVaga()
1. Busca vagas FREE ordenadas por andar asc + número asc
2. Se user.pcd === true → prioriza vaga tipo DISABLED
3. Se user.role === 'VIP' → prioriza vaga tipo VIP
4. Fallback → primeira STANDARD disponível (andar mais baixo)
TypeScript — spots.service.ts
async function atribuirMelhorVaga(lotId, userRole, userPcd) {
  const vagas = await prisma.spot.findMany({
    where: { lotId, status: 'FREE' },
    orderBy: [{ floor: 'asc' }, { number: 'asc' }],
  });
  if (userPcd) {
    const pcd = vagas.find(v => v.type === 'DISABLED');
    if (pcd) return pcd;
  }
  if (userRole === 'VIP') {
    const vip = vagas.find(v => v.type === 'VIP');
    if (vip) return vip;
  }
  return vagas.find(v => v.type === 'STANDARD') ?? vagas[0];
}

2. Cálculo de Tarifa

// fare.utils.ts — calcularTarifa()
horas = Math.max(minutos / 60, 1) // mínimo cobrado: 1 hora
valor = horas × pricePerHour
se ocupação > 80%: valor ×= 1.20 // +20% preço dinâmico

Property-based Tests — fast-check

fare.utils.spec.ts
import * as fc from 'fast-check';
import { calcularTarifa } from '../src/sessions/fare.utils';

describe('Cálculo de tarifa — propriedades', () => {
  it('nunca cobra menos que 1 hora', () => {
    fc.assert(fc.property(
      fc.integer({ min: 1, max: 59 }),   // minutos < 1h
      fc.float({ min: 5, max: 50 }),      // preço/hora
      (minutes, price) => {
        const fare = calcularTarifa(minutes, price, 0.5);
        expect(fare).toBeGreaterThanOrEqual(price);
      }
    ));
  });

  it('aplica +20% quando ocupação > 80%', () => {
    fc.assert(fc.property(
      fc.integer({ min: 60, max: 480 }),
      fc.float({ min: 5, max: 50 }),
      fc.float({ min: 0.81, max: 1.0 }),
      (minutes, price, occupancy) => {
        const base    = calcularTarifa(minutes, price, 0.5);
        const dynamic = calcularTarifa(minutes, price, occupancy);
        expect(dynamic).toBeCloseTo(base * 1.2, 2);
      }
    ));
  });

  it('tarifa é sempre positiva', () => {
    fc.assert(fc.property(
      fc.integer({ min: 1, max: 1440 }),
      fc.float({ min: 1, max: 100 }),
      fc.float({ min: 0, max: 1 }),
      (minutes, price, occupancy) => {
        expect(calcularTarifa(minutes, price, occupancy)).toBeGreaterThan(0);
      }
    ));
  });
});

Diferenciais

O que torna o ParkMe especial

💡
Dor real resolvida
A narrativa começa com um problema genuíno que milhões de brasileiros enfrentam diariamente. O projeto tem propósito claro, não é apenas um CRUD.
⚡
Arquitetura event-driven
WebSocket para atualizações instantâneas de vagas em todos os clientes (mobile + dashboard). Redis para cache de sessões ativas. Eventos como cidadãos de primeira classe.
🗺️
Mapa SVG com Pathfinding BFS
Não é só mostrar um mapa — o algoritmo BFS calcula a rota animada da entrada até a vaga, andar por andar. Uma funcionalidade de produto real, não só demonstração técnica.
🖥️
Dashboard operacional completo
Next.js 16 com 10 páginas funcionais: mapa em tempo real, KPIs de analytics, busca por placa/vaga com debounce, badge de notificações real e modal de entrada que exibe a vaga atribuída em destaque.
📱
Full-stack integrado mobile + backend + dashboard
Três clientes (React Native, Next.js e Swagger) consumindo a mesma API com comunicação bidirecional em tempo real. Projetados juntos, não como projetos separados.
💳
Pagamento real com webhook
MercadoPago com Pix e cartão, confirmação via webhook assíncrono. A vaga só é liberada após confirmação real de pagamento — exatamente como em produção.
🧪
Property-based testing com fast-check
Além de testes unitários clássicos, propriedades matemáticas do cálculo de tarifa são verificadas contra milhares de inputs aleatórios. Nível avançado de qualidade de código.
🇧🇷
Contexto brasileiro nativo
MercadoPago como gateway, Pix como método de pagamento, placa no formato Mercosul, português como língua padrão. Um projeto feito para o mercado nacional.
