Problema & Solução

Por que o ParkMe existe

Você chega em um shopping lotado, anda em círculos por 10 minutos procurando uma vaga, finalmente encontra uma — e quando volta, passa mais 15 minutos tentando lembrar em qual setor deixou o carro. O aplicativo do estacionamento (se existir) não atualiza em tempo real, o painel de vagas está quebrado, e você perdeu a reunião.

ParkMe resolve isso.

🗺️
Mapa ao vivo
Visualize todas as vagas em tempo real com atualização via WebSocket. Verde = livre, vermelho = ocupado.
🧭
Pathfinding BFS
Algoritmo BFS traça a rota animada até a vaga mais próxima dentro do estacionamento.
💳
Pagamento integrado
Pix ou cartão via MercadoPago direto no app. Vaga só é liberada após confirmação.
🔔
Notificações push
Alerta 15 minutos antes do tempo acabar. Geolocalização detecta quando você está a 200m.
⚡
Event-driven
Arquitetura assíncrona com Redis + BullMQ para filas de notificações e jobs agendados.

WebSocket Events

Eventos em tempo real via Socket.io — namespace /parking

🅿️
spot_occupied Server → Client
Emitido quando uma vaga muda de FREE para OCCUPIED. Todos os clientes na room do estacionamento recebem.
{ "spotId": "spot_042", "sector": "B", "floor": 1, "number": 42 }
✅
spot_free Server → Client
Emitido após confirmação de pagamento. A vaga volta a ficar disponível no mapa.
{ "spotId": "spot_042", "sector": "B", "floor": 1, "number": 42 }
⏰
session_expiring Server → Client (privado)
Enviado 15 minutos antes de completar o tempo pago. Somente para o motorista da sessão.
{ "sessionId": "ses_001", "minutesLeft": 15, "totalAmount": 18.50 }
📊
lot_occupancy Server → Client (broadcast)
Broadcast a cada 30 segundos com o percentual de ocupação do estacionamento.
{ "lotId": "lot_001", "total": 120, "occupied": 97, "percentage": 80.8 }
📱 Exemplo — Cliente React Native 
// hooks/useWebSocket.ts
import { useEffect } from 'react';
import { io } from 'socket.io-client';
import { useParkingStore } from '../store/parkingStore';

export function useWebSocket(lotId: string) {
  const { updateSpot } = useParkingStore();

  useEffect(() => {
    const socket = io('http://api:3000/parking', {
      auth: { token: getToken() },
    });

    // Entra na room do estacionamento
    socket.emit('join_lot', { lotId });

    // Atualiza o mapa SVG quando uma vaga muda
    socket.on('spot_occupied', (data) => updateSpot(data.spotId, 'OCCUPIED'));
    socket.on('spot_free',     (data) => updateSpot(data.spotId, 'FREE'));

    return () => socket.disconnect();
  }, [lotId]);
}

Regras de Negócio

Algoritmos, fórmulas e transições de estado

1. Atribuição Automática de Vagas

O algoritmo escolhe a melhor vaga disponível no momento da entrada, seguindo prioridade:
// Pseudocódigo de atribuição
1. Andar 1 → menor número de vaga (prioridade física)
2. Tipo VIP → somente se user.role === 'VIP'
3. Tipo PCD → somente se user.pcd === true
4. Fallback → próxima STANDARD disponível
TypeScript — assignSpot() 
async function assignSpot(lotId: string, user: User): Promise<Spot> {
  // Busca vagas livres ordenadas por andar e número
  const available = await prisma.spot.findMany({
    where: { lotId, status: 'FREE' },
    orderBy: [{ floor: 'asc' }, { number: 'asc' }],
  });

  // PCD tem prioridade se usuário marcado
  if (user.pcd) {
    const pcdSpot = available.find(s => s.type === 'DISABLED');
    if (pcdSpot) return pcdSpot;
  }

  // VIP tem prioridade se role for VIP
  if (user.role === 'VIP') {
    const vipSpot = available.find(s => s.type === 'VIP');
    if (vipSpot) return vipSpot;
  }

  // Padrão: primeira vaga STANDARD disponível
  return available.find(s => s.type === 'STANDARD');
}

2. Cálculo de Tarifa

// Fórmula base
valor = (minutos / 60) × pricePerHour

// Mínimo cobrado: 1 hora
valor = Math.max(valor, pricePerHour)

// Precificação dinâmica
se ocupação > 80%: valor = valor × 1.20
TypeScript — calculateFare() 
export function calculateFare(
  minutes: number,
  pricePerHour: number,
  occupancyRate: number
): number {
  // Converte minutos para horas, mínimo 1h
  const hours = Math.max(minutes / 60, 1);

  let fare = hours * pricePerHour;

  // Aplica acréscimo dinâmico se estacionamento cheio
  if (occupancyRate > 0.8) {
    fare = fare * 1.2;
  }

  // Arredonda para 2 casas decimais
  return parseFloat(fare.toFixed(2));
}

Property-based Tests — fast-check

pricing.spec.ts 
import * as fc from 'fast-check';
import { calculateFare } from '../src/sessions/fare.utils';

describe('Cálculo de tarifa — propriedades', () => {

  it('nunca deve cobrar menos que 1 hora', () => {
    fc.assert(
      fc.property(
        fc.integer({ min: 1, max: 59 }),   // minutos < 1h
        fc.float({ min: 5, max: 50 }),     // preço/hora
        (minutes, pricePerHour) => {
          const fare = calculateFare(minutes, pricePerHour, 0.5);
          expect(fare).toBeGreaterThanOrEqual(pricePerHour);
        }
      )
    );
  });

  it('deve aplicar +20% quando ocupação > 80%', () => {
    fc.assert(
      fc.property(
        fc.integer({ min: 60, max: 480 }),  // 1-8h
        fc.float({ min: 5, max: 50 }),
        fc.float({ min: 0.81, max: 1.0 }),  // ocupação > 80%
        (minutes, pricePerHour, occupancy) => {
          const base    = calculateFare(minutes, pricePerHour, 0.5);
          const dynamic = calculateFare(minutes, pricePerHour, occupancy);
          expect(dynamic).toBeCloseTo(base * 1.2, 2);
        }
      )
    );
  });

  it('tarifa deve ser sempre positiva', () => {
    fc.assert(
      fc.property(
        fc.integer({ min: 1, max: 1440 }),
        fc.float({ min: 1, max: 100 }),
        fc.float({ min: 0, max: 1 }),
        (minutes, price, occupancy) => {
          const fare = calculateFare(minutes, price, occupancy);
          expect(fare).toBeGreaterThan(0);
        }
      )
    );
  });

});

Diferenciais

O que torna o ParkMe especial

💡
Dor real resolvida
A narrativa começa com um problema genuíno que milhões de brasileiros enfrentam diariamente. O projeto tem propósito claro, não é apenas um CRUD.
⚡
Arquitetura event-driven
WebSocket para atualizações instantâneas de vagas + BullMQ para filas de notificações + Redis para cache de sessões ativas. Eventos como cidadãos de primeira classe.
🗺️
Mapa SVG com Pathfinding BFS
Não é só mostrar um mapa — o algoritmo BFS calcula a rota animada da entrada até a vaga, andar por andar. Uma funcionalidade de produto real, não só demonstração técnica.
📱
Full-stack integrado mobile + backend
React Native + NestJS com comunicação bidirecional em tempo real. O app mobile e a API foram projetados juntos, não como projetos separados.
💳
Pagamento real com webhook
MercadoPago com Pix e cartão, confirmação via webhook assíncrono. A vaga só é liberada após confirmação real de pagamento — exatamente como em produção.
🧪
Property-based testing com fast-check
Além de testes unitários clássicos, propriedades matemáticas do cálculo de tarifa são verificadas contra milhares de inputs aleatórios. Nível avançado de qualidade.
🌍
Contexto brasileiro nativo
MercadoPago como gateway, Pix como método de pagamento, endereços brasileiros no seed, português como língua padrão. Um projeto feito para o mercado nacional.
